Лабораторная работа 1: Подготовка окружения для создания консольного PHP-приложения, работающего с SQLite
Установка SQLite

Для начала работы с нашим проектом, вам необходимо установить SQLite на своем компьютере. 
Убедитесь, что у вас есть следующие файлы для работы с этой СУБД, и ознакомьтесь с интерактивным режимом, 
используя стандартного консольного клиента sqlite3.

Создание shell-скрипта, работающего с SQLite
Мы разработаем командный файл для записи статистики о запусках нашей программы и вывода этой информации на экран. 
Этот файл будет называться self-logger.bat для Windows или self-logger.sh для Mac/Linux. Следующие данные будут 
сохранены в нашей SQLite-базе данных:
Имя программы: self-logger.bat
Количество запусков: 10
Первый запуск: 2021.09.04 16:54

---------------------------------------------
User      | Date
---------------------------------------------
andrey    | 2021.09.03 16:52
---------------------------------------------
...

При первом запуске программа автоматически создаст БД с необходимой структурой.

Установка PHP
Настройте среду для работы с PHP в режиме командной строки (CLI), без необходимости установки веб-сервера.

Настройка GitHub и Git
Зарегистрируйтесь на GitHub и сделайте форк своего репозитория из репозитория.

Установите Git и выполните первичную настройку на вашей локальной машине 
(git configure --global user.name, git configure --global user.email).

Склонируйте форкнутый репозиторий из своего профиля GitHub на вашу локальную машину (git clone). 
Ваш удаленный репозиторий будет привязан к источнику с именем "origin".

Добавьте привязку к оригинальному репозиторию под именем "upstream" 
(git remote add upstream https://github.com/andpop-mrsu/402_XXX_XX.git).

Создайте ветку Task01 в вашем локальном репозитории и работайте в ней.

Начало первого проекта
Создайте каталог Task01 и добавьте файл README.md (в формате Markdown) с описанием вашего проекта.

Напишите PHP-скрипт task1.php, который будет выводить на экран инструкцию к игре "Угадай число". 
Описание игры для вашего варианта находится в файле specification_01_nn.md.

Отправка задания на проверку
Добавьте файл README_logger.md с инструкцией по запуску командного файла self-logger.bat или self-logger.sh.

Сделайте коммит в ветке Task01 
(в этой папке должны быть файлы: README.md, README_logger.md, self-logger.bat/self-logger.sh и task1.php).

Отправьте созданный коммит в ваш "форкнутый" репозиторий на GitHub (git push origin).

Создайте Pull Request для ветки Task01 из вашего репозитория в репозиторий.

Ваше задание будет считаться выполненным после закрытия Pull Request в репозитории.

После принятия задания преподавателем, не забудьте синхронизировать свой локальный репозиторий
с оригинальным (git pull upstream master) и репозиторий в вашем профиле на GitHub (git push origin master).

Задача для первого блока - Вариант 2

Вам предстоит разработать программу для игры "Угадай число" (guess-number). Ваша задача заключается в следующем:

Компьютер загадывает число в диапазоне от 1 до максимального числа, заданного в настройках.
Игрок должен угадать это число за несколько попыток, при этом максимальное количество попыток задано в настройках.
После каждой попытки компьютер сообщает, было ли введенное число больше или меньше загаданного.
Ваша программа также должна сохранять информацию о датах и результатах всех игр, а также о каждой попытке,
сделанной во время игры. Для каждой игры в базе должна храниться следующая информация:

Дата игры
Имя игрока
Максимально возможное число
Загаданное компьютером число
Исход игры (угадал/не угадал)
Запись попыток в формате: номер попытки | предложенное число | ответ компьютера
В программе должны быть реализованы следующие режимы:

Новая игра.
Вывод списка всех сохраненных игр.
Вывод списка всех игр, в которых победил человек.
Вывод списка всех игр, в которых человек проиграл.
Вывод статистики по игрокам. Для каждого игрока нужно посчитать количество побед
